## 詳細建議

### 1. 資料矛盾問題（高優先）

- `test_summary.md` FM-004 備註寫「移動對話框尚未實作」，但截圖已證實有實作
- `test_summary.md` FM-005 備註寫「無確認對話框」，但已修復
- **需同步更新 summary 備註**

### 2. FM-008 測試不足（高優先）

- 僅 1 張截圖，步驟只有「檢查 UI 元件」
- 建議至少 3 張截圖：Drop Zone → 拖拉中 → 上傳完成

### 3. 缺少負向測試（高優先）

建議新增：
- 錯誤密碼登入 → 應顯示錯誤
- 重複資料夾名稱 → 應拒絕
- 刪除不存在的檔案 → 應 404
- 上傳超大檔案 → 應有錯誤提示

### 4. 建立 iteration_log.md（中優先）

記錄每個 Round 的失敗原因與修復方式。

### 5. description.md 模板統一（中優先）

所有案例應包含：
- 修正紀錄區塊
- 截圖對照表
- 明確的斷言條件

### 6. 自動化改進（低優先）

- 寫腳本自動生成 JSON/summary
- 加入 response time 記錄
