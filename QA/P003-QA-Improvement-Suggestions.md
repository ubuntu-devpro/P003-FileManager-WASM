| 案例 | 截圖數量 |

| FA002 多網域權限 | 2 張 |

| FA004 Session驗證 | **0 張**（無截圖需求）|
| FM001 資料夾瀏覽 | 3 張 |
| FM002 新增資料夾 | 4 張 |
| FM003 重新命名 | 5 張 |

| FM005 刪除 | 4 張 |

### 2. 📝 Description.md 內容不完整

**發現：**
- 部分 description 缺少「實際結果」欄位（有些只有預期結果）
- 缺少截圖與步驟的對照說明
- 缺少「修正紀錄」區塊（記錄發現問題與修復經過）

**建議：統一 description.md 模板：**
```markdown
## 測試資訊
[完整資訊表]

## 功能說明
[描述功能]

## 測試步驟
1. 步驟一
2. 步驟二

## 預期結果
- 結果一

## 實際結果
[填入實際觀察]

## 截圖對照
| 步驟 | 截圖 | 說明 |
|------|------|------|
| 1 | 001_xxx.png | 描述 |

## 修正紀錄
- YYYY-MM-DD：發現 XX 問題，已修正

## 狀態判定
✅ PASS / ❌ FAIL
```

### 3. 📊 缺少截圖失敗的歷史紀錄

**發現：**
- FM002/003/004 早期截圖是失敗的（dialog 沒出現）
- Round1~Round5 的演進過程沒有記錄
- `screenshots/dialogs_v5/` 是失敗時期的截圖，但未說明

**建議：**
- 在 QA 目錄下建立 `history/` 或 `iteration_log.md`
- 記錄每個 Round 的發現與修正
- 失敗的截圖移到 `screenshots/failed/` 並標註原因

### 4. 🔢 截圖數量異常的案例需要關注

### 5. 📁 QA 目錄結構可以優化

**現況：**
```
QA/
├── Test0407-Round1/
├── Test0407-Round2/
├── Test0407-Round3/
├── Test0407-Round4/
└── Test0407-Round5/
```

**建議：**
- 每個 Round 的資料過多，可改為：
```
QA/
├── reports/              # 最終報告
├── iterations/           # 每次測試的 iteration
│   ├── Round1/
│   ├── Round2/
│   └── ...
├── history/             # 失敗截圖與問題記錄
└── templates/          # description.md 模板
```
