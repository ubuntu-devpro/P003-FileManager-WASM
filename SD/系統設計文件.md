## 3. 關鍵設計決策

### 3.1 無 SignalR（Blazor WASM）

相較於 P002 的 Blazor Server：
- 優點：無電路連線問題，Playwright 自動化穩定
- 缺點：客戶端需自行管理 auth state
- 結論：WASM 更適合自動化優先的專案

### 3.2 JWT vs Session Cookie

- 使用 JWT Bearer Token
- Token 存於 client localStorage
- HttpClient 自動附加 `Authorization: Bearer {token}`
- Server 無狀態，適合水平擴展

### 3.3 網域過濾在 API 層

- `FilesController` 從 JWT claims 取得 `domain`
- `GetRootPathForUser()` 回傳對應的根目錄
- `FileService` 做路徑安全檢查（`../` 遍歷防護）

### 3.4 無資料庫

- 使用者寫死在 `JwtService` 程式碼中
- 未來可擴充至資料庫（Entity Framework Core）

## 4. 安全性設計

### 4.1 路徑遍歷防護

```csharp
private bool IsPathAllowed(string path)
{
    var fullPath = Path.GetFullPath(path);
    var rootFull = Path.GetFullPath(RootPath);
    return fullPath.StartsWith(rootFull);
}
```

### 4.2 JWT 驗證

所有 `/api/*` 端點（除 `/api/auth/login`）皆需附上有效 JWT，否則回 `401 Unauthorized`。

### 4.3 網域隔離

一般使用者的 API 請求只能觸及 `/home/devpro/data/{domain}/`，無法跨越。
